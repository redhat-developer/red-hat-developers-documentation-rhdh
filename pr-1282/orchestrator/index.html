Installing Red Hat Developer Hub with the Orchestrator plugin

Install, configure, and manage Red Hat Developer Hub with the Orchestrator plugin using Helm or the Red Hat Developer Hub Operator

Abstract

As a cluster administrator, you can install Red Hat Developer Hub (RHDH) with the Orchestrator plugin enabled.
  • 1. Installing Red Hat Developer Hub with the Orchestrator plugin using Helm
    • 1.1. Resource limits for installing Red Hat Developer Hub with the Orchestrator plugin

You can install Red Hat Developer Hub (RHDH) using one of the following methods:

  • The Red Hat Developer Hub Operator
  • The Red Hat Developer Hub Helm chart

1. Installing Red Hat Developer Hub with the Orchestrator plugin using Helm

You can install Red Hat Developer Hub with Orchestrator by using Helm. This method provides flexibility and customization for managing workflow orchestration infrastructure.

1.1. Resource limits for installing Red Hat Developer Hub with the Orchestrator plugin

When installing Red Hat Developer Hub (RHDH) with the Orchestrator plugin using Helm, the chart defines default CPU and memory limits for the SonataFlowPlatform component.

These limits are enforced by the cluster so that pods do not exceed their allocated resources.

  1. Default resource limits
ResourceDefault value

CPU limits

500m

Memory limits

1Gi

You can override these values in the following ways:

  • With values.yaml

  • With --set flags

    1. Override defaults with values.yaml as shown in the following example: 

      orchestrator:
  sonataflowPlatform:
    resources:
      limits:
        cpu: "500m"
        memory: "1Gi"

    2. Override with --set as shown in the following example: 

      helm upgrade --install <release_name>  openshift-helm-charts/redhat-developer-hub \
  --set orchestrator.sonataflowPlatform.resources.requests.cpu=500m \
  --set orchestrator.sonataflowPlatform.resources.requests.memory=128Mi \
  --set orchestrator.sonataflowPlatform.resources.limits.cpu=1 \
  --set orchestrator.sonataflowPlatform.resources.limits.memory=2Gi
