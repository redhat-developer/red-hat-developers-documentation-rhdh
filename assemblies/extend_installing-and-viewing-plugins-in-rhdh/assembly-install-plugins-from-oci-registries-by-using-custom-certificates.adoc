= Install plugins from OCI registries by using custom certificates

:context: install-plugins-from-oci-registries-by-using-custom-certificates

[role="_abstract"]
In {product-very-short}, you can install OCI plugins stored in an internal OCI artifact registry served over HTTPS with customer CA certificates. The following example shows a configuration in the `dynamic-plugins.yaml` file:

[source,terminal,subs="+attributes,+quotes"]
----
includes:
- dynamic-plugins.default.yaml

plugins:
- disabled: false
  package: oci://reg.example.com:5000/myplugin:v0.0.1
----

== Prerequisites
* You have a corporate CA bundle or a set of custom container-registry TLS certificates that the system should trust.
+
[NOTE]
====
You can create a CA bundle from a set of CA certificates manually, by concatenating them into a single file, as follows:

[source,terminal]
----
# Concatenate CA certificates
cat registry.crt intermediate.crt corporate-root.crt > ca-bundle.crt

# Validate
openssl verify -CAfile ca-bundle.crt registry.crt
----
====
