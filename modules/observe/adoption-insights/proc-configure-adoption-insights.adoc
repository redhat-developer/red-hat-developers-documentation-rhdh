= Configuring the Adoption Insights plugin in {product}

[role="_abstract"]
Configure the Adoption Insights plugin parameters through the config map to customize plugin behavior and settings.

The Adoption Insights plugin is preinstalled and enabled on a {product-short} instance by default.
You can disable or enable the Adoption Insights plugin and change other parameters by configuring the `dynamic-plugins.yaml` file.

.Procedure

. If you previously used the Developer Preview of the Adoption Insights plugin and enabled the plugin manually, remove those earlier plugins as they are no longer required, and Adoption Insights is enabled by default.

.. In your `dynamic-plugins.yaml` file, remove the references to the following plugins:
+
* `red-hat-developer-hub-backstage-plugin-adoption-insights`
* `red-hat-developer-hub-backstage-plugin-adoption-insights-backend`
* `red-hat-developer-hub-backstage-plugin-analytics-module-adoption-insights`

.. Enable the `backstage-community-plugin-analytics-provider-segment` plugin by setting `disabled` to `false`:
+
[source,yaml]
----
plugins:
  - package: ./dynamic-plugins/dist/backstage-community-plugin-analytics-provider-segment
    disabled: false
----
