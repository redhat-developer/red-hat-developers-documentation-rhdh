= Enable {azure-short} Monitor metrics

[role="_abstract"]
Enable managed Prometheus monitoring for your {aks-name} ({aks-short}) cluster to collect metrics and monitor {product-short} performance through {azure-short} Monitor.

.Prerequisites

* You have an {aks-short} cluster.
* You have the {azure-short} CLI installed.

.Procedure

* To enable managed Prometheus monitoring, use the `--enable-azure-monitor-metrics` option with either the `az aks create` or `az aks update` command:
+
[source,bash]
----
$ az aks create/update --resource-group <your-ResourceGroup> --name <your-Cluster> --enable-azure-monitor-metrics
----
+
This command installs the metrics add-on, which gathers https://learn.microsoft.com/en-us/azure/azure-monitor/metrics/prometheus-metrics-overview[Prometheus metrics].

.Verification

* In the {azure-short} portal, navigate to *Monitoring -> Insights* to view the monitoring results. For more information, see https://learn.microsoft.com/en-us/azure/azure-monitor/platform/monitor-azure-resource[Monitor {azure-short} resources with {azure-short} Monitor].

.Additional resources

* You can access metrics from both the Managed Prometheus service and {azure-short} Monitor through the {azure-short} Managed Grafana service. For more information, see https://learn.microsoft.com/en-us/azure/azure-monitor/metrics/azure-monitor-workspace-manage?tabs=azure-portal#link-a-grafana-workspace[Link a Grafana workspace].
* By default, Prometheus uses the minimum ingesting profile. You can customize the default settings through custom configuration. For more information, see https://learn.microsoft.com/en-us/azure/azure-monitor/containers/prometheus-metrics-scrape-default[Default Prometheus metrics configuration in {azure-short} Monitor] and https://learn.microsoft.com/en-us/azure/azure-monitor/containers/prometheus-metrics-scrape-configuration?tabs=CRDConfig%2CCRDScrapeConfig[Customize scraping of Prometheus metrics in {azure-short} Monitor managed service for Prometheus].
