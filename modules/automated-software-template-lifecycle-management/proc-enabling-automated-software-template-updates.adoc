= Enabling automated template updates

To automate the synchronization of changes from Software Templates to your repositories, you must configure the plugin in your backend settings and ensure your entities contain the required metadata.

.Prerequisites

* You have configured GitHub or GitLab integrations in your `{product-very-short} app-config.yaml` file.
* Your scaffolded entities include the `spec.scaffoldedFrom` field referencing the source template.
* Your entities include the `backstage.io/managed-by-location` annotation pointing to a valid GitHub or GitLab URL.

.Procedure

. Open your `{product-very-short} app-config.yaml` file.
. Enable the pull request (PR) feature by adding the following configuration:
+
[source,yaml]
----
scaffolder:
  pullRequests:
    templateUpdate:
      enabled: true
----
. Optional: Enable notifications to alert entity owners when a PR is created:
+
[source,yaml]
----
scaffolder:
  notifications:
    templateUpdate:
      enabled: true
----
. Restart the {product} backend to apply the changes.

.Verification

. Update the version of a source template in its repository.
. Navigate to a repository scaffolded from that template.
. Confirm that a new pull request named `[component-name]/template-upgrade-v[version]` exists.
