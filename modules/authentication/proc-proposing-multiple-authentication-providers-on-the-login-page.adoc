= Proposing multiple authentication providers on the login page

When multiple authentication providers are configured in your {product-short} instance, the login page presents users with options to choose their preferred authentication method.

.Prerequisites
* You have configured multiple authentication providers in your `{my-app-config-file}` file. For example, you might have set up both {rhbk-brand-name} ({rhbk}) and GitHub as authentication providers.

.Procedure
* Configure multiple providers in the `signInPage` section of your `{my-app-config-file}` like this:
+
[source,yaml]
----
signInPage:
  - oidc
  - github 
----

.Verification
. Navigate to the {product-short} login page.
. On the login page, you see tiles representing each configured authentication provider.
. Click on the tile corresponding to your preferred authentication provider.
. You are redirected to the selected authentication provider's login page to complete the authentication process.
