= Enabling service-to-service authentication by using JWKS tokens

You can user JSON Web Key Sets (JWKS) tokens to enable service-to-service authentication.

JWKS tokens are significantly more secure than static tokens as they are signed and issued by a trusted Identity Provider, making them a much better choice for production deployments.

This approach offers strong security benefits via three key characteristics:

Asymmetric encryption::
JWKS uses a pair of shared keys: one public, one private, instead of a single shared static token.
The Identity Provider signs the JWT with its private key, then {product-short} verifies it using the public key fetched from the JWKS endpoint.
This means {product-short} never has access to the private signing key, reducing the risk of compromise.

Easy key rotation::
The Identity Provider can rotate signing keys regularly without requiring changes to {product-short}, when the JWKS endpoint is updated.
This minimizes downtime and enhances security.

Ability to validate claims::
JWKS tokens include claims like issuer and audience, which can be verified by {product-short} to ensure the token is from a trusted source and prevents it from being used in unintended contexts.

.Prerequisites


* You have administrative access to configure {product-short} in your OpenShift cluster.
* You have a JWKS endpoint available from your Identity Provider that {product-short} can access.
* You have configured your external service to obtain a JWT from your Identity Provider and include it in the `Authorization` header of requests to {product-short}.

.Procedure

. Add the JWKS URL to your `{my-app-config-file}` {product-short} configuration ile.
+
[source,yaml,subs="+attributes,+quotes"]
----
backend:
  auth:
    externalAccess:
      - type: jwks
        options:
          url: _<your_jwks_endpoint>_
          algorithm: RS256
          issuer: http://your-idp.example.com
          audience: management
          subjectPrefix: your-prefix
----

url::
Enter the URL of your JWKS endpoint, such as `pass:c,a,q[http://your-idp.example.com/.well-known/jwks.json]`.

algorithm::
(Optional) Enter the signing algorithm used by your Identity Provider (commonly RS256).

issuer::
(Optional) Enter the expected issuer claim in the token  `iss` field.

audience::
(Optional) Enter the expected audience claim in the token `aud` field.

subjectPrefix::
(Optional) Define a prefix to add to the subject claim, and to display in the Audit Log for debugging and tracking purposes.
