:_mod-docs-content-type: SNIPPET
. Mirror the catalog index image to your internal registry. The `install-dynamic-plugins` init container pulls this image at startup for plugin discovery.
+
[source,terminal,subs="+quotes,+attributes"]
----
$ skopeo copy docker://registry.redhat.io/rhdh/plugin-catalog-index:{product-version} docker://_<target_registry>_/rhdh/plugin-catalog-index:{product-version}
----

. Create a ConfigMap with a `registries.conf` file that redirects the init container to your internal registry:
+
[source,yaml,subs="+quotes"]
----
apiVersion: v1
kind: ConfigMap
metadata:
  name: rhdh-mirror-conf
data:
  rhdh-registries.conf: |
    [[registry]]
    prefix = "registry.redhat.io/rhdh"
    location = "_<target_registry>_/rhdh"
----

. Mount the ConfigMap in the `install-dynamic-plugins` init container by adding the following to your `{product-custom-resource-type}` CR:
+
[source,yaml,subs="+attributes"]
----
spec:
  application:
    extraFiles:
      configMaps:
        - name: rhdh-mirror-conf
          key: rhdh-registries.conf
          mountPath: /etc/containers/registries.conf.d
          containers:
            - install-dynamic-plugins
----
+
[NOTE]
====
Cluster-level mirroring resources such as `ImageDigestMirrorSet` or `ImageContentSourcePolicy` do not apply to the init container because it uses `skopeo` directly to pull images.
====
