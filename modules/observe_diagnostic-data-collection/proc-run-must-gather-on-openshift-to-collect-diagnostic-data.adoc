= Run must-gather on OpenShift to collect diagnostic data

[role="_abstract"]
Collect diagnostic data from {product-very-short} deployments on {ocp-short}.

.Prerequisites

* You have deployed {product} on {ocp-short}.
* Cluster administrator access or `RBAC` permissions for resource inspection.
* Authenticated {openshift-cli} ({ocp-short}).

.Procedure

. Run must-gather:
+
[source,terminal,subs="+attributes"]
----
$ oc adm must-gather --image=registry.access.redhat.com/rhdh/rhdh-must-gather-rhel9:{product-version}
----
+
Output: `./must-gather.local.<timestamp>`
+
[NOTE]
====
The must-gather tool automatically detects and collects data from all {product-very-short} instances in the cluster, regardless of whether they were deployed using the Operator or Helm chart.

For advanced scenarios, you can limit data collection:

. Skip Helm-based deployments: `-- /usr/bin/gather --without-helm`
. Skip Operator-based deployments: `-- /usr/bin/gather --without-operator`
====

.Verification

* Confirm that the output directory exists:
+
[source,terminal]
----
$ ls must-gather.local.<timestamp>/
----
+
The directory contains subdirectories for each deployment method found:
+
--
* `operator/` - Data from Operator-based deployments
* `helm/` - Data from Helm-based deployments
--
