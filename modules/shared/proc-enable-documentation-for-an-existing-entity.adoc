= Enable documentation for an existing entity

[role="_abstract"]
You can add TechDocs documentation to a component that is already registered in the {product-short} software catalog but does not yet have documentation configured.

.Prerequisites

* An existing entity is registered in the {product-short} software catalog.
* You have write access to the entity's source repository.

.Procedure

. Add the `backstage.io/techdocs-ref` annotation to the `catalog-info.yaml` file in your repository:
+
[source,yaml]
----
apiVersion: backstage.io/v1alpha1
kind: Component
metadata:
  name: my-component
  annotations:
    backstage.io/techdocs-ref: dir:.
spec:
  type: service
  lifecycle: production
  owner: group:default/my-team
----

. Create an `mkdocs.yml` file in the repository root:
+
[source,yaml]
----
site_name: My Component Documentation
nav:
  - Home: index.md
----
+
[NOTE]
====
{product-short} automatically adds the `techdocs-core` plugin to `mkdocs.yml` if it is missing.
====

. Create a `docs/` directory in the repository root containing at least an `index.md` file with your documentation content in Markdown.

. Commit, push, and merge the changes.

.Verification

. In the {product-short} software catalog, navigate to your component.
. Verify that a *Docs* tab appears with your documentation.
+
[NOTE]
====
The *Docs* tab might not appear immediately. {product-short} refreshes catalog entities periodically, which can take up to 45 minutes. To trigger a refresh sooner, click the refresh button on the entity's *Overview* tab.
====
