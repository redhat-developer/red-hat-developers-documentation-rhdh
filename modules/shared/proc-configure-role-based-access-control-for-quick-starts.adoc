= Configure role-based access control for quick starts

[role="_abstract"]
You can control which users see specific quick start guides by configuring role-based access control (RBAC) for quick starts in your {my-app-config-file} file.

.Prerequisites
* You have configured RBAC in {product-very-short}.

.Procedure

. Enable RBAC in your {my-app-config-file} file:
+
[source,yaml]
----
permission:
  enabled: true
----
+
When RBAC is enabled, the system determines user roles based on permissions:
+
** Users with `policy.entity.create` permission are assigned the admin role.
** Users without this permission are assigned the developer role.
+
[NOTE]
====
If RBAC is disabled (`permission.enabled: false`) or not configured, users are assumed to be platform engineers setting up {product} ({product-very-short}) and are assigned the admin role.
====

. Configure quick start items with role assignments in your {my-app-config-file} file:
+
[source,yaml]
----
app:
  quickstart:
    - title: 'Platform Configuration'
      titleKey: steps.platformConfiguration.title
      roles: ['admin']
      # Only admins see this
    - title: 'Getting Started as Developer'
      titleKey: steps.gettingStarted.title
      roles: ['developer']
      # Only developers see this
    - title: 'Universal Welcome Guide'
      titleKey: steps.universalWelcome.title
      roles: ['admin', 'developer']
      # Both user roles see this
----
+
The following roles are supported:
+
admin:: Platform engineers, administrators, and users with elevated permissions
developer:: Regular developers and users with standard permissions
+
[NOTE]
====
Quick start items without a `roles` property default to the admin role. Items can specify multiple roles, and users see quick start items that match their assigned role.
====
