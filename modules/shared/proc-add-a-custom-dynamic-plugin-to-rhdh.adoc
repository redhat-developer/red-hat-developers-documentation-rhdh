= Add a custom dynamic plugin to {product}

[role="_abstract"]
Use this procedure to add custom dynamic plugins to {product} by updating the dynamic-plugins.yaml configuration file.

.Procedure
* To add your custom dynamic plugins to {product}, update the `dynamic-plugins.yaml` file by using the following configuration that the `npx @red-hat-developer-hub/cli@latest plugin package` command generates:
+
[source,yaml]
----
plugins:
  - package: oci://quay.io/_<user_name>_/entity-feedback-plugin:0.5.0
    disabled: false
  - package: oci://quay.io/_<user_name>_/entity-feedback-plugin-backend:0.6.0
    disabled: false
----
+
[NOTE]
====
Ensure that your container images are publicly accessible, or that you have configured a pull secret in your environment. A pull secret provides {product} with credentials to authenticate pulling your plugin container images from a container registry.
====
