= Match the platform appearance to your company brand colors and styles

[role="_abstract"]
Apply custom color schemes, logos, and visual elements to the {product} interface using the new front-end system so the platform reflects your organization's brand identity.

The new front-end system uses `@backstage/frontend-defaults` to provide modern {backstage} theme architecture.

.Prerequisites

* You have installed the `@backstage/frontend-defaults` package.
* You have access to the `packages/app/src/App.tsx` file.
* You have access to the `{my-app-config-file}` file.

.Procedure

. To enable the custom theme, import the `rhdhThemeModule` from the theme plugin alpha module in your `packages/app/src/App.tsx` file:
+
[source,typescript]
----
import { createApp } from '@backstage/frontend-defaults';
import { rhdhThemeModule } from '@red-hat-developer-hub/backstage-plugin-theme/alpha';

export default createApp({
  features: [
    rhdhThemeModule,
    // Additional features
  ],
});
----
+
[NOTE]
====
The `rhdhThemeModule` applies custom themes by overriding the extensions of the theme plugin. For advanced theme customization, you can create a custom theme module that extends the base theme configuration.
====

. To set the application title and branding, configure extension settings in your `{my-app-config-file}` file:
+
[source,yaml]
----
app:
  title: My Company Catalog
  baseUrl: http://localhost:3000

organization:
  name: My Company
----

.Verification

. Start your application in the new front-end system mode.
. Verify that the interface displays your custom theme.
. Check that theme colors match your brand identity specifications.
