= Configure persona-specific homepage layouts for role-based experiences

[role="_abstract"]
Attach homepage configurations to RBAC groups to provide role-specific layouts. Users automatically see homepage cards appropriate to their group's responsibilities.

.Prerequisites

* You installed and configured the homepage backend plugin.
* You defined RBAC groups for each user persona (for example, `team/developers`, `team/data-scientists`, `team/managers`).
* You have administrative access to modify the `{my-app-config-file}` configuration file.

.Procedure

. Identify your user personas and their corresponding RBAC groups.
+
Create a mapping table that associates each user role with an RBAC group:
+
[options="header"]
|===
|Persona |RBAC Group |Purpose
|Developer |`team/developers` |Software developers creating and deploying applications
|Data Scientist |`team/data-scientists` |Data scientists working with ML models and pipelines
|Manager |`team/managers` |Engineering managers monitoring team health
|Platform Engineer |`team/platform-engineering` |Platform engineers managing {product-very-short}
|===
+
[NOTE]
====
Replace example group names with your organization's actual RBAC group names.
====

. Verify the RBAC groups exist in your {product-very-short} configuration.
+
Verify RBAC group existence by using the RBAC web UI or by reviewing your RBAC configuration file.
Ensure you defined all groups referenced in your persona mapping before proceeding.

. Open your `{my-app-config-file}` configuration file.

. Define the homepage layout for each persona in the `homepage.backend.defaultWidgets.mountPoints` section.
+
For example, configure a developer persona homepage with search, quick access, templates, and starred entities.
+
[source,yaml]
----
homepage:
  backend:
    defaultWidgets:
      mountPoints:
        - mountPoint: home.page/cards
          importName: SearchBar
          config:
            layouts:
              xl: { w: 10, h: 1, x: 1 }
              lg: { w: 10, h: 1, x: 1 }
            visibilityRules:
              groups: ['team/developers']

        - mountPoint: home.page/cards
          importName: QuickAccessCard
          config:
            layouts:
              xl: { w: 7, h: 8 }
              lg: { w: 7, h: 8 }
            props:
              title: Developer Quick Access
            visibilityRules:
              groups: ['team/developers']

        - mountPoint: home.page/cards
          importName: TemplateSection
          config:
            layouts:
              xl: { w: 12, h: 5 }
              lg: { w: 12, h: 5 }
            visibilityRules:
              groups: ['team/developers']

        - mountPoint: home.page/cards
          importName: CatalogStarredEntitiesCard
          config:
            layouts:
              xl: { w: 5, h: 4, x: 7 }
              lg: { w: 5, h: 4, x: 7 }
            visibilityRules:
              groups: ['team/developers']
----

. Attach each homepage configuration to its corresponding RBAC group by using the `visibilityRules.groups` field.
+
The `groups` field accepts an array of RBAC group names.
Users in any of the listed groups will see the card.
+
[NOTE]
====
The front-end positions cards according to layout coordinates (`x`, `y`, `w`, `h`), not array order. The order in the configuration array does not affect rendering position.
====
. Configure a default fallback homepage for users who are not members of any persona-specific RBAC group.
+
Provide a minimal safe default with essential cards:
+
[source,yaml]
----
        - mountPoint: home.page/cards
          importName: SearchBar
          config:
            layouts:
              xl: { w: 10, h: 1, x: 1 }
            visibilityRules: {}  # Visible to all users

        - mountPoint: home.page/cards
          importName: QuickAccessCard
          config:
            layouts:
              xl: { w: 6, h: 6 }
            visibilityRules: {}  # Visible to all users
----

. Save the configuration file and restart {product}:
+
[source,terminal]
----
$ kubectl rollout restart deployment/<rhdh-deployment-name>
----
+
Replace `<rhdh-deployment-name>` with the name of your {product-very-short} deployment.

.Verification

. Log in to {product} as a test user who belongs to one of the configured RBAC groups.

. Verify that the homepage displays only the cards from that user's persona configuration.

. Log in as a test user from a different RBAC group and verify they see a different set of homepage cards.

. Log in as a test user who is not a member of any persona group and verify they see the fallback homepage configuration.
