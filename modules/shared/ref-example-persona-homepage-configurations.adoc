= Example persona homepage configurations

[role="_abstract"]
Ready-to-use homepage configuration examples for common user personas in {product-very-short}.
Copy and adapt these examples to quickly implement persona-based homepages for your organization.

== Developer persona homepage

Software developers creating and deploying applications need quick access to templates, catalog entities, and development documentation.

Recommended cards for developers:

* SearchBar: Quickly find catalog entities and documentation
* QuickAccessCard: Links to common developer workflows
* TemplateSection: Browse and create new components from templates
* CatalogStarredEntitiesCard: Quick access to frequently used components
* FeaturedDocsCard: Relevant developer documentation

Complete configuration:

[source,yaml]
----
homepage:
  backend:
    defaultWidgets:
      mountPoints:
        - mountPoint: home.page/cards
          importName: SearchBar
          config:
            layouts:
              xl: { w: 10, h: 1, x: 1 }
              lg: { w: 10, h: 1, x: 1 }
              md: { w: 10, h: 1, x: 1 }
              sm: { w: 12, h: 1 }
            visibilityRules:
              groups: ['team/developers']

        - mountPoint: home.page/cards
          importName: QuickAccessCard
          config:
            layouts:
              xl: { w: 7, h: 8 }
              lg: { w: 7, h: 8 }
              md: { w: 7, h: 8 }
              sm: { w: 12, h: 8 }
            props:
              title: Developer Quick Access
            visibilityRules:
              groups: ['team/developers']

        - mountPoint: home.page/cards
          importName: TemplateSection
          config:
            layouts:
              xl: { w: 12, h: 5 }
              lg: { w: 12, h: 5 }
              md: { w: 12, h: 5 }
              sm: { w: 12, h: 5 }
            visibilityRules:
              groups: ['team/developers']

        - mountPoint: home.page/cards
          importName: CatalogStarredEntitiesCard
          config:
            layouts:
              xl: { w: 5, h: 4, x: 7 }
              lg: { w: 5, h: 4, x: 7 }
              md: { w: 5, h: 4, x: 7 }
              sm: { w: 12, h: 4 }
            visibilityRules:
              groups: ['team/developers']

        - mountPoint: home.page/cards
          importName: FeaturedDocsCard
          config:
            layouts:
              xl: { w: 12, h: 4 }
              lg: { w: 12, h: 4 }
              md: { w: 12, h: 4 }
              sm: { w: 12, h: 4 }
            visibilityRules:
              groups: ['team/developers']
----

== Data scientist persona homepage

Data scientists working with ML models and data pipelines need access to data science tools, data catalog, and model registries.

Recommended cards for data scientists:

* SearchBar: Find data catalog entities and ML models
* QuickAccessCard: Links to Jupyter notebooks, data sources, ML platforms
* MarkdownCard: Custom links to data science tools and documentation
* CatalogStarredEntitiesCard: Frequently used datasets and models

Complete configuration:

[source,yaml]
----
homepage:

    defaultWidgets:
      mountPoints:
        - mountPoint: home.page/cards
          importName: SearchBar
          config:
            layouts:
              xl: { w: 10, h: 1, x: 1 }
              lg: { w: 10, h: 1, x: 1 }
            visibilityRules:
              groups: ['team/data-scientists']

        - mountPoint: home.page/cards
          importName: QuickAccessCard
          config:
            layouts:
              xl: { w: 6, h: 8 }
              lg: { w: 6, h: 8 }
            props:
              title: Data Science Tools
            visibilityRules:
              groups: ['team/data-scientists']

          importName: MarkdownCard
          config:
            layouts:

== Manager and team lead persona homepage

Recommended cards for managers:

* SearchBar: Find team components and projects
* QuickAccessCard: Links to team dashboards and reporting tools

== Hybrid users with multiple personas

Users who belong to multiple RBAC groups inherit homepage configurations from all groups.
Use OR logic in visibility rules to show cards to users in any of several groups.

For example, show the TemplateSection card to users who are either developers or team leads:

[source,yaml]
----
- mountPoint: home.page/cards
  importName: TemplateSection
  config:
    layouts:
      xl: { w: 12, h: 5 }
    visibilityRules:
      groups: ['team/developers', 'team/team-leads']
----

Users who are members of both `team/developers` and `team/managers` see homepage cards from both persona configurations, creating a combined experience.
