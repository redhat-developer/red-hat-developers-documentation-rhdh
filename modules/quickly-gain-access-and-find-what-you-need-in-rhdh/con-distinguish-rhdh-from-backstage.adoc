= Distinguish {product-very-short} from {backstage}

While {product-very-short} is built on the open-source {backstage} project, it offers distinct advantages for enterprise environments. {product-very-short} is a production-ready, supported distribution that removes the maintenance burden associated with the upstream project.

.Shared foundation

Both platforms provide the same core developer portal capabilities, including the Software Catalog, Software Templates (Scaffolder), TechDocs, and global search.

.Key differences

The following points highlight the key differences to help you understand additional values of {product-very-short}:

[cols="1,2,2", options="header"]
|===
| Feature
| {backstage}
| {product-very-short}

| *Maintenance Model*
| DIY (Do It Yourself). You maintain the codebase and perform upgrades.
| Zero-code solution. {company-name} manages the base image and updates.

| *Support*
| Community support only.
| 24x7 Enterprise Support.

| *Plugin Architecture*
| Requires recompilation to add plugins.
| Dynamic plugins allow changes without recompiling.

| *Security*
| Self-managed security updates.
| Enterprise-grade security, RBAC, and rapid CVE resolution.

| *Deployment*
| Helm-based installer.
| Kubernetes Operator-based installer (OpenShift, EKS, AKS, GKE).
|===

== Selection criteria

* *Choose {backstage} if:* You have a dedicated platform team to maintain code, manage security updates, and build custom integrations yourself.
* *Choose {product-very-short} if:* You need a production-ready, supported platform with built-in security, RBAC, and a zero-code maintenance model.
