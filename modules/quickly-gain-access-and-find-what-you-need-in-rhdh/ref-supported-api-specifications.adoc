= Supported API specifications in {product-very-short}

To make sure your API definitions display correctly in {product-very-short}, use a supported specification format. You can review the primary formats and their typical use cases to maintain compatibility with the platform interface.

* OpenAPI: The standard format for defining REST APIs, formerly known as Swagger.
* AsyncAPI: A format used for event-driven architectures.
* GraphQL: A format used for GraphQL API schemas.

[NOTE]
====
Support for specific formats, such as AsyncAPI or GraphQL, depends on your instance configuration.
====
