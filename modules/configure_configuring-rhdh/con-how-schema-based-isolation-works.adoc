= How schema-based isolation works

[role="_abstract"]
When you set `pluginDivisionMode` to `schema`, {product-short} stores all plugin data in a single PostgreSQL database and uses separate schemas for isolation. 

Each plugin gets its own schema instead of its own database, which eliminates the need for the `CREATEDB` privilege and reduces connection overhead.

The {product} {product-short} backend manages the following:

Schema discovery::
The backend scans the primary database to identify existing schemas that match the internal naming convention of the active plugins.

Dynamic schema creation::
If a required schema is missing—and the service account has sufficient permissions—the backend automatically executes a `CREATE SCHEMA` command.

Versioned migrations::
Once a schema is established, the backend applies versioned SQL migration scripts to build or update the internal table structures.
