= Benefits of consolidated database provisioning

[role="_abstract"]
By using schema-based isolation, platform engineers and database administrators can enhance security and simplify resource management. This approach ensures data remains partitioned and accessible only to authorized services within the environment.

Schema-based isolation provides several operational advantages:

Resource efficiency::
Consolidating data into a single database instance reduces the number of active connection pools, lowering the CPU and memory consumption on the PostgreSQL server.

Reduced administrative privileges::
The Red Hat Developer Hub service account does not require high-level permissions, such as `SUPERUSER` or `CREATEDB`. The account only needs sufficient privileges to create and manage objects within the designated schemas of the primary database.

Centralized maintenance::
Database management tasks, including backups, point-in-time recovery, and security patching, are centralized to a single instance rather than distributed across multiple databases.

Compliance with corporate policies::
Many enterprise environments prohibit applications from dynamically creating new databases. Schema-based isolation allows the Hub to function within a pre-provisioned database environment while maintaining strict data separation.
