= PostgreSQL permissions matrix for service accounts

[role="_abstract"]
Minimum database privileges required for the Red Hat Developer Hub service account to manage logical namespaces when using schema-based isolation.

These permissions allow the application to manage logical namespaces within a single database without requiring superuser or database-creation privileges.

== Database-level privileges

The Red Hat Developer Hub service account requires the following privileges at the database level to establish connectivity and initialize the schema architecture.

[title="Required database-level permissions"]
[cols="1,3", options="header"]
|===
| Privilege | Requirement

| `CONNECT`
| Mandatory. Required for the Red Hat Developer Hub backend to establish a session with the PostgreSQL instance.

| `CREATE`
| Mandatory. Required for the application to automatically generate plugin-specific schemas, such as `backstage_plugin_catalog`, during the initial startup or after adding new plugins.

| `TEMPORARY`
| Recommended. Allows the creation of temporary tables to support complex query operations and data processing.
|===



== Schema-level privileges for pre-existing environments

In environments where security policies require database administrators to manually pre-provision schemas, the service account must be granted the following privileges for each specific plugin schema.

[title="Required schema-level permissions"]
[cols="1,3", options="header"]
|===
| Privilege | Requirement

| `USAGE`
| Required for the service account to access, view, and perform operations on objects within the designated schema.

| `CREATE`
| Required for the service account to define and modify tables, indexes, and sequences within the schema namespace.
|===

[source,sql]
----
-- Example: Granting minimum database-level permissions
GRANT CONNECT, CREATE, TEMPORARY ON DATABASE rhdh_db TO rhdh_user;
----
