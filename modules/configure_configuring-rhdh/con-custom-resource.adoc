= {product-custom-resource-type} custom resource

[role="_abstract"]
When you use the {product} Operator, the `{product-custom-resource-type}` Custom Resource (CR) is the central deployment configuration. It defines how the Operator creates and manages the {product-short} instance, and references the ConfigMaps and Secrets that contain your configuration files.

The following fields control key aspects of the deployment:

`spec.application.appConfig.configMaps`::
References ConfigMaps containing your `{my-app-config-file}` files.

`spec.application.dynamicPluginsConfigMapName`::
References the ConfigMap containing the `dynamic-plugins.yaml` file.

`spec.application.extraFiles`::
Mounts additional ConfigMaps, Secrets, or PVCs as files in the {product-short} container.

`spec.application.extraEnvs`::
Injects environment variables from ConfigMaps, Secrets, or inline values.

`spec.application.route`::
Controls Route or Ingress configuration.

`spec.database`::
Controls the built-in PostgreSQL database (enable or disable).

`spec.deployment.patch`::
Applies a strategic merge patch to the Deployment resource for advanced customization.
