= Schema routing and namespace isolation

[role="_abstract"]
Prevent data collisions in multi-plugin environments by using the plugin divider to isolate database traffic. The divider automatically appends schema prefixes to database requests, ensuring data integrity across shared infrastructure.

This process ensures that data operations remain scoped to the correct logical boundary. Without the plugin divider, the Hub would attempt to create all tables in the default `public` schema, leading to table name collisions and security vulnerabilities between different plugins.
