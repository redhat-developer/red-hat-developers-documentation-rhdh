= Software Catalog default column identifiers

[role="_abstract"]
Use these column IDs to configure `exclude` or `include` options.

.Default catalog table columns
[cols="1,3",options="header"]
|===
|Column ID |Description

|`name`
|Entity display name and primary catalog identifier.

|`owner`
|Team or individual responsible for the entity (from `spec.owner`).

|`type`
|Entity type classification (from `spec.type`). Examples: `service`, `library`, `website`.

|`lifecycle`
|Entity lifecycle stage (from `spec.lifecycle`). Examples: `production`, `experimental`, `deprecated`.

|`description`
|Entity purpose and summary (from `metadata.description`).

|`tags`
|Labels for categorization and filtering (from `metadata.tags`).

|`namespace`
|Entity namespace (from `metadata.namespace`).

|`system`
|Parent system for the entity (from `spec.system`).

|`createdAt`
|Entity creation timestamp. Empty if unavailable.
|===

[NOTE]
====
Use `exclude` to hide any of these columns. Use `include` to show only specific columns in the order you list them.
====
