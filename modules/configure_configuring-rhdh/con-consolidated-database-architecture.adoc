= Consolidated database architecture

[role="_abstract"]
Use schema isolation to consolidate multiple plugin database connections into a single connection to a primary database. This configuration reduces connection overhead and streamlines resource management for database administrators.

The backend uses a routing mechanism to direct data to specific namespaces. This configuration creates a structured environment where each plugin operates within its own logical boundary:
* Catalog data is stored in a dedicated catalog schema.
* Scaffolder data is stored in a dedicated scaffolder schema.
* Authentication data is stored in a dedicated auth schema.
