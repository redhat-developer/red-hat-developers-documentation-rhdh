= Enable plug-in division mode schema 

[role="_abstract"]
Configure a single PostgreSQL database for {product-short} by defining the database host, user credentials, and plugin division mode in your dynamic configuration file.


[NOTE]
====
By default, {product-very-short} automatically creates the required schemas. If a database user lacks `Create Schema` privileges, add `ensureSchemaExists: false` to the database configuration. The database administrator must create all required schemas.
====

.Procedure

. Open the OpenShift web console and navigate to the *ConfigMaps* section in the Red Hat Developer Hub project.

. Edit the `app-config.yaml` file.

. Navigate to the `backend.database` section and set the `pluginDivisionMode` parameter to `schema`.

. Ensure the `client` is set to `pg` and verify the connection details for your single database:
+
[source,yaml]
----
backend:
  database:
    client: pg
    pluginDivisionMode: schema
    connection:
      host: ${POSTGRES_HOST}
      port: ${POSTGRES_PORT}
      user: ${POSTGRES_USER}
      password: ${POSTGRES_PASSWORD}
----

. Save the changes to the ConfigMap.

. Restart the Red Hat Developer Hub deployment to apply the new configuration.
