= Available scorecard metric IDs

[role="_abstract"]
Use the following metric IDs when disabling scorecard metrics per entity or globally.

== Filecheck metrics

`filecheck._<key>_`:: Verifies that the configured file exists in the repository. The metric ID suffix is derived from the key you define in `scorecard.plugins.filecheck.files`.

== GitHub metrics

`github.open_prs`:: Number of open pull requests in GitHub.

== Jira metrics

`jira.open_issues`:: Number of open issues in Jira.

== OpenSSF metrics

`openssf.binary_artifacts`:: Determines if the project has generated executable (binary) artifacts in the source repository.
`openssf.branch_protection`:: Determines if the default and release branches are protected with branch protection settings.
`openssf.cii_best_practices`:: Determines if the project has an OpenSSF (formerly CII) Best Practices Badge.
`openssf.ci_tests`:: Determines if the project runs tests before pull requests are merged.
`openssf.code_review`:: Determines if the project requires human code review before pull requests are merged.
`openssf.contributors`:: Determines if the project has a set of contributors from multiple organizations.
`openssf.dangerous_workflow`:: Determines if the project's GitHub Action workflows avoid dangerous patterns.
`openssf.dependency_update_tool`:: Determines if the project uses a dependency update tool.
`openssf.fuzzing`:: Determines if the project uses fuzzing.
`openssf.license`:: Determines if the project has defined a license.
`openssf.maintained`:: Determines if the project is actively maintained.
`openssf.packaging`:: Determines if the project is published as a package.
`openssf.pinned_dependencies`:: Determines if the project has declared and pinned the dependencies of its build process.
`openssf.sast`:: Determines if the project uses static code analysis.
`openssf.security_policy`:: Determines if the project has published a security policy.
`openssf.signed_releases`:: Determines if the project cryptographically signs release artifacts.
`openssf.token_permissions`:: Determines if the project's workflows follow the principle of least privilege.
`openssf.vulnerabilities`:: Determines if the project has open, known unfixed vulnerabilities.
