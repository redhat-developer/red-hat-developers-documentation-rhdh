= Install and configure the Keycloak plugin

[role="_abstract"]
The Keycloak backend plugin integrates Keycloak into {product-short}, enabling synchronization of Keycloak users and groups in a realm.

.Prerequisites

* Your {product-short} application is installed and running.

.Procedure

. Enable the Keycloak plugin by setting the `disabled` property to `false` in your dynamic plugins configuration:
+
[source,yaml]
----
global:
  dynamic:
    includes:
      - dynamic-plugins.default.yaml
    plugins:
      - package: ./dynamic-plugins/dist/backstage-community-plugin-catalog-backend-module-keycloak-dynamic
        disabled: false
----

. Set the following environment variables to configure the Keycloak connection:
+
* `KEYCLOAK_BASE_URL`
* `KEYCLOAK_LOGIN_REALM`
* `KEYCLOAK_REALM`
* `KEYCLOAK_CLIENT_ID`
* `KEYCLOAK_CLIENT_SECRET`

. Optional: Configure a schedule in the `app-config.yaml` file:
+
[source,yaml]
----
catalog:
  providers:
    keycloakOrg:
      default:
        schedule:
          frequency: { minutes: 1 }
          timeout: { minutes: 1 }
          initialDelay: { seconds: 15 }
----
+
[NOTE]
====
If you have made any changes to the schedule in the `app-config.yaml` file, then restart to apply the changes.
====

. Optional: Override the default Keycloak query parameters in the `app-config.yaml` file:
+
[source,yaml]
----
catalog:
  providers:
    keycloakOrg:
      default:
        userQuerySize: 500
        groupQuerySize: 250
----

.Verification

* Verify that Keycloak users and groups are synchronized into the {product-short} catalog.
