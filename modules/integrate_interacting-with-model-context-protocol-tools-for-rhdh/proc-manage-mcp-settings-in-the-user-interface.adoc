= Manage MCP settings in the user interface

[role="_abstract"]
Manage Model Context Protocol (MCP) server connections and personal authentication tokens directly from the {ls-brand-name} chat interface. {backstage} administrators configure the available MCP servers in the {backstage} `{my-app-config-file}` file.

You can use the *MCP settings* panel to review server status, see a number of tools available for each server, and enable or disable specific tools for your session.

.Procedure

. In the {ls-brand-name} chat box, click the menu icon (*Chatbot options*) in the header.
. Select *MCP settings*.
. In the *MCP settings* panel, perform any of the following actions:
** Review server status: View whether available MCP servers are enabled or disabled.
** View available tools: View the count of specific tools available for each server under *Status*.
** Configure personal tokens: 
*** To add or update a token, click the *Edit* icon, enter your Personal Access Token in the field, and click *Save*. The system validates the token automatically.
*** To remove or replace a token, click the *Edit* icon and select *Forget Token*.
** Enable or disable servers: Use the toggles to select which servers provide tools for your chat session.
+
[NOTE]
====
You must configure a token before you can enable a server that requires authentication.
====

.Verification

* Confirm the status message displayed below the token field:
** *Connection successful*: The token is valid and verified.
** *Invalid credentials*: The token is incorrect or the connection failed.
* Submit a query in the chat that requires an MCP tool, such as requesting catalog resources or server information, to confirm the tools are active.
* If a server is disabled, confirm that {ls-short} searches its resources or documentation to provide a response.
