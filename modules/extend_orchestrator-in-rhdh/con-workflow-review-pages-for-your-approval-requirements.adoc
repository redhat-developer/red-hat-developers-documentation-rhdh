= Workflow review pages for your approval requirements

[role="_abstract"]
You can replace the default Orchestrator review page with your own component to match organizational standards, show warnings or require acknowledgment before run, or integrate with design systems in {product}.

The Orchestrator plugin displays a review page before executing workflows, showing form data for verification. While the default review page works for most use cases, some organizations need custom layouts, in-page warnings or approval prompts, or integration with existing UI components.

Custom review pages are optional. If you do not implement a custom review component, the Orchestrator continues to use the default review page without any impact on functionality.

Use a custom review page when you need to:

* Display workflow data in a specific layout that matches your organization's documentation or approval standards
* Show warnings or apply client-side checks before calling the execute callback (or before the workflow runs).
* Include additional context, help text, or documentation for reviewers
* Integrate with custom UI component libraries or design systems

Custom review pages maintain full compatibility with existing workflows. The same workflow definitions, schemas, and data structures work with both default and custom review pages. You can switch between default and custom review pages without modifying your workflow configurations.
