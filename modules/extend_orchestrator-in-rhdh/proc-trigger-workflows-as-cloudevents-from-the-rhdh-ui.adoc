= Trigger workflows as CloudEvents from the {product-very-short} UI

[role="_abstract"]
Trigger event-driven workflows from the {product-very-short} UI by publishing CloudEvents to Apache Kafka.
This method provides asynchronous workflow execution, where the workflow starts when the Kafka broker delivers the CloudEvent to the SonataFlow workflow engine.

.Prerequisites

* Kafka connectivity is configured for the Orchestrator. For more information, see xref:configure-kafka-connection-for-cloudevent-workflows_{context}[].
* An event-type workflow is deployed and visible in the Orchestrator plugin.

.Procedure

. In the {product-very-short} UI, navigate to the Orchestrator plugin.

. In the workflows list, locate the workflow you want to trigger.

. Click the *Run as Event* button next to the workflow.
+
[NOTE]
====
The *Run as Event* button is only visible when Kafka connectivity is configured and the workflow supports event-driven execution.
====

. If the workflow requires input data, complete the workflow input form.
+
The form fields correspond to the workflow's input schema.
Values you provide are included in the CloudEvent data payload.

. Click *Submit* to send the CloudEvent to Kafka.
+
The {product-very-short} UI sends a CloudEvent to the configured Kafka broker with the workflow input data.
The workflow starts when the Kafka broker delivers the event to the SonataFlow engine.

. Monitor the workflow execution status.
+
After submitting the CloudEvent, one of two outcomes occurs:
+
--
Immediate execution:: If the workflow starts before the UI timeout period, the UI navigates to the workflow instance detail page, where you can monitor execution progress.

Delayed execution:: If the workflow has not started when the UI timeout expires, the UI displays an informational message indicating that the event was sent to Kafka as a `kafkaEvent`.
The UI then navigates to the workflow runs list, where the workflow instance will appear when execution starts.
--

. If the workflow does not start immediately, locate the workflow in the workflow runs list.
+
The workflow instance is displayed in the list when the Kafka broker delivers the CloudEvent and the workflow engine starts execution.
Depending on Kafka broker latency and workflow engine processing time, this can take several seconds.

.Troubleshooting

If the workflow does not appear in the workflow runs list after several minutes:

* Verify that the Kafka broker is running and accessible.
* Check the orchestrator-backend logs for errors related to Kafka message publishing.
* Confirm that the workflow is configured to consume CloudEvents from the correct Kafka topic.
* Verify that the CloudEvent type matches the workflow's event type configuration.
