= Execute the database copy script to securely extract and transfer local catalog data

[role="_abstract"]
Execute the database copy script to migrate data from the local PostgreSQL instance to your external PostgreSQL server.

.Prerequisites

* You have installed the link:https://www.postgresql.org/docs/current/app-pgdump.html[`pg_dump`] and link:https://www.postgresql.org/docs/current/app-psql.html[`psql`] utilities on your local machine.
* You have PostgreSQL user privileges to make a full dump of local databases.
* You have PostgreSQL admin privileges to create databases on the external server.
* You have configured port forwarding for the local PostgreSQL database pod:
+
[source,terminal]
----
$ oc port-forward -n <your_namespace> <pgsql_pod_name> <forward_to_port>:<forward_from_port>
----
+
Where:
+
`<pgsql_pod_name>`:: The name of the PostgreSQL pod, in the format `backstage-psql-<deployment_name>-<_index>`.
`<forward_to_port>`:: The port of your choice to forward PostgreSQL data to.
`<forward_from_port>`:: The local PostgreSQL instance port, such as `5432`.

.Procedure

. Create a copy of the following `db_copy.sh` script and edit the details based on your configuration:
+
[source,bash]
----
#!/bin/bash

to_host=<db_service_host>
to_port=5432
to_user=postgres

from_host=127.0.0.1
from_port=15432
from_user=postgres

allDB=("backstage_plugin_app" "backstage_plugin_auth" "backstage_plugin_catalog" "backstage_plugin_permission" "backstage_plugin_scaffolder" "backstage_plugin_search")

for db in ${!allDB[@]};
do
  db=${allDB[$db]}
  echo Copying database: $db
  PGPASSWORD=$TO_PSW psql -h $to_host -p $to_port -U $to_user -c "create database $db;"
  pg_dump -h $from_host -p $from_port -U $from_user -d $db | PGPASSWORD=$TO_PSW psql -h $to_host -p $to_port -U $to_user -d $db
done
----
+
Where:
+
`to_host`:: The destination hostname, for example `<db_instance_name>.rds.amazonaws.com`.
`to_port`:: The destination port, such as `5432`.
`to_user`:: The destination server username, for example `postgres`.
`from_host`:: The source hostname, such as `127.0.0.1`.
`from_port`:: The source port number, such as the `<forward_to_port>` variable.
`from_user`:: The source server username, for example `postgres`.
`allDB`:: Database names to import, in double quotes separated by spaces.

. Run the script to copy the data to the destination database:
+
[source,terminal]
----
$ /bin/bash TO_PSW=<destination_db_password> /path/to/db_copy.sh
----
+
Replace `<destination_db_password>` with the password to connect to the destination database.

. When the data copy is complete, stop port forwarding.

[NOTE]
====
For more information about handling large databases and using compression tools, see the link:https://www.postgresql.org/docs/current/backup-dump.html#BACKUP-DUMP-LARGE[Handling Large Databases] section on the PostgreSQL website.
====
