= Sizing requirements for cluster resource provisioning

[role="_abstract"]
Platform engineers can calculate infrastructure requirements for {product} deployments using established sizing guidelines for components, databases, and scaling across different deployment levels.

The sizing approach considers four deployment scales: small-scale (up to 5K entities, 50 users), mid-scale (5-20K entities, 50-150 users), large-scale (20-50K entities, 150-400 users), and enterprise-scale (50-150K entities, 400-800 users).

Resource allocation covers compute requirements for the {product-short} application, PostgreSQL database sizing, and infrastructure overhead for supporting services and high availability configurations.

== PostgreSQL performance tuning parameters

For external PostgreSQL deployments, configure performance parameters based on allocated memory:

[cols="25%,25%,25%,25%", frame="all", options="header"]
|===
|Sizing legend
|Memory
|`shared_buffers`
|`effective_cache_size`

|Small-scale
|8 GiB
|2 GB
|4 GB

|Mid-scale
|16 GiB
|4 GB
|8 GB

|Large-scale
|32 GiB
|8 GB
|16 GB

|Enterprise-scale
|64 GiB
|16 GB
|32 GB
|===

Set `shared_buffers` to approximately 1/4 of the allocated database memory and `effective_cache_size` to approximately 1/2 of the allocated database memory.

[NOTE]
====
Using an external PostgreSQL instance is recommended for production deployments.
